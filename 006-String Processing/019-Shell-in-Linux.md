# üêö Shell in Linux

A **Shell** in Linux is a **command-line interpreter** that acts as a bridge between the **user** and the **kernel** of the operating system.  
It takes commands from the user, interprets them, and passes them to the operating system for execution.

---

## üîë Key Functions of Shell
- Interprets and executes user commands.
- Provides **scripting capabilities** for automation.
- Manages **I/O redirection** and **piping**.
- Supports **environment variables** and **command history**.

---

## üõ†Ô∏è Types of Shells in Linux
| Shell | Command | Description |
|-------|----------|-------------|
| **Bourne Shell (sh)** | `sh` | The original UNIX shell. |
| **Bash (Bourne Again Shell)** | `bash` | Most common and default shell on Linux. |
| **C Shell (csh)** | `csh` | Syntax similar to C programming. |
| **Korn Shell (ksh)** | `ksh` | Combines features of sh and csh. |
| **Z Shell (zsh)** | `zsh` | Advanced shell with many features. |
| **Fish Shell** | `fish` | User-friendly shell with auto-suggestions. |
